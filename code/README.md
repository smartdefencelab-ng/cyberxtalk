# CyberXTalk Code Workspace

This folder holds implementation work for CyberXTalk.

The implementation is organized to correspond with the four paper tracks in `../papers/`.

## Structure

```text
code/
├── 01-framework-context-aware-multilingual-incident-intelligence/
├── 02-taxonomy-data-schema-informal-cyber-reports/
├── 03-context-aware-classification-and-triage/
└── 04-multilingual-low-resource-nlp-incident-understanding/
```

## Purpose

Each implementation folder should contain the practical system artifacts needed to support the corresponding paper:

- APIs
- schemas
- experiments
- notebooks
- pipelines
- tests
- demos
- evaluation scripts

## Safety Rule

Do not commit secrets, private datasets, model checkpoints, generated audio, large runtime artifacts, or anything that could expose sensitive implementation details.
