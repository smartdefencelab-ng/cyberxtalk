# Paper 2: Cyber Incident Taxonomy and Data Schema

## Working Title

**A Context-Aware Taxonomy and Data Schema for Cyber Incident Reporting in Low-Resource African Digital Environments**

## Core Idea

This paper defines the structured representation layer for CyberXTalk: how informal human cybercrime reports should be organized into analyzable cyber incident objects.

## Research Gap

Informal user reports are not logs. They are human narratives containing incomplete context, emotional descriptions, missing technical details, and mixed language.

A useful cyber incident intelligence system needs a taxonomy and schema before classification, triage, dashboards, or escalation can work reliably.

## Contribution

- Cyber incident taxonomy for informal citizen/SME reports.
- Data schema for pre-CERT incident reporting.
- Fields for incident type, victim category, attack channel, affected platform, evidence type, financial loss, urgency, and escalation path.
- Foundation for dataset creation, annotation, classification, and evaluation.

## Possible Schema Dimensions

- Incident type
- Report source
- Victim category
- Attack channel
- Affected platform/institution
- Loss type
- Evidence type
- Narrative language
- Urgency/severity
- Recommended escalation path
- Response status

## Possible Research Questions

1. What data fields are necessary to transform informal cybercrime reports into structured cyber incident records?
2. How can a taxonomy capture cyber incidents reported by non-technical users?
3. How can the schema support classification, triage, escalation, and intelligence reporting?

## Notes

This paper gives CyberXTalk its scientific backbone. Without a schema, the AI layer becomes ungrounded.
